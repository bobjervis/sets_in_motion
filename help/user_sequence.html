Sequence Editor

The sub-window that you can use to view or edit a whole sequence is called a Sequence Editor.  It appears in the left-hand pane of the main window.

The image on the right shows a sample Sequence Editor containing an new, barely begun sequence.  The top field is a one-line comment field.  You can right anything you like in there.  It will show in the Dance Editor and may help you identify it in a long list of sequences, or you may use it to contain information on difficulty.

The Created field gives the date when the sequence was created.

The Modified field contains the date of the last modification.

The Created With field identifies the program and version used to create the sequence.  Note that for any sequence created with this program, you will see "SiMv1.0alpha3" in that field.  Future releases of the program will update that value.  That way, if there is any question about the compatibility of an older file with a future version, this information might help.

The Level field is the level of the Sequence.  All calls in the sequence must have a level less than or equal to this value.  If you select a blank value, any calls at all can be used in the sequence regardless of level.

The rest of the window is taken up by the Call Area.  Each call takes one line of the display.  Currently, long lines will not wrap around, so you must resize the window if the space is not wide enough for your call.

The Call Area itself is white and is large enough to hold the existing calls and the current call being typed (as in the example).  The rest of the area is available to hold suggested completions for the current call.  Since the user has typed "heads s" the program has found quite a few possible calls that start with S. 

You can click on any of the calls in the call area that have their text in black.  If this were a resolved sequence there would also appear a grey 'home' after the last call.  You cannot select the 'home' lines.  They are there to indicate the start and end of the sequence.  Once the sequence resolves and the trailing 'home' string appears, you must change some existing call in the sequence to alter the position.  The program will automatically add or remove the 'home' string as appropriate.

When no traiiling 'home' string, you can click on the blank line immediately below the last call in order to lengthen the sequence.

Note that if you type a call that does not work, the word 'Failed' in red will appear to the left of the call.  In addition, an explanation will be written in red characters below the call itself.  In some cases, a call will be legal, but the program will insert a comment string in orange.  This is a flow warning and you can ignore it if you like.  At this point, the warnings are experimental as I am trying to figure out exactly what conditions are worth tagging.

Note that if the call failed and the error text below the call is written inside a solid yellow background, this indicates a program bug, rather than a user error.  In general, if the error text is on a white background, the reason the call did not work is that, according to the definitions in the program, this call is not allowed from the current position.

Please report all program bugs to me.  Also report any instance in which you disagree with the program's decision to mark a call as failed.

Unless the call has failed because of some grammatical mistake (like a typo), you will notice that a small button with a question mark appears on the line to the far left of the call.  This is a drilldown button.  Whenever you see a button like this, clicking on it will open up a Call Analysis page in the right-hand side of the main window.  This view will give you a step-by-step guide through the call.  Depending on what is happening in the call, there may be further drilldown buttons in the analysis page as well.  See the documentation of the Call Analysis Browser for more details.

Note that you can cut-copy-paste text from a single line to or from these text fields to other fields in this program or in any other Windows application.  Pasting large multi-line blocks of text from another program may not produce very good results in Sets in Motion.

In addition to positioning the text cursor in a particular call, when you click on a call in the call area (including either 'home' string) you will open an Animation screen in the right-hand side of the main window.  The blue triangle that appears to the left of a call in the Sequence Editor indicates where the Animation screen is currently showing.  See the documentation of Animation for more details.

If you press the Insert key when the cursor is in any call text, you will immediately insert a new, blank call into the sequence at the line where the cursor is currently located.

If you right-click over a call, you will be given a menu with two choices:

  • Insert a call  Shifts all calls from the one under the mouse cursor to the bottom of the sequence down one and inserts a new blank call into the current line.
  • Delete this call  Shift all calls below this line up by one and erases the current call.
