Grammar Editor

The sub-window that gives you the top-level view of a definitions file is called a Grammar Editor (since this allows you to define vocabulary, synonyms and valid phrases for use in calls).  The Grammar Editor is shown as an outline.  The image at right shows how this outline appears when you have closed all the outline items except the top-level filename line itself.  As you can see, small boxes contain either a minus or plus sign.  Each such box can be clicked to alter how the outline is displayed. 

Clicking on a minus box will close that item and contract everything under it.  The box itself then changes into a plus box.  Clicking on a plus box will expand the underlying outline items and change the box to a minus box.

If no further outline items exist under a given level, no box will be drawn next to the item.  Currently, each synonym, definition and formation are the lowest levels of this outline.

Currently there is nothing you can do with synonyms, formations or dancer designations.  This will be remedied in a future release.

When you expand outline, if there is not enough room, scrollbars will be included to allow you to adjust the portion of the list that is visible.

The definitions are labeled with the first phrase used to call that definition.  As you change the phrases for a call, the list of definitions will not be re-sorted until you re-start the program.

If you right-click on the title 'Definitions' it offers you a menu with a single item on it: Create New Definition.  Selecting that menu item will create a new definition that is completely empty and ready to be filled in.  It is added to the end of the list of definitions in the current file.  The definition will be opened in a Definition Editor in the left-hand side of the main window, covering the outline view.  You can return to the outline by clicking on the tab marked either 'Definitions' or 'My Definitions' (depending on which you want to see).

If you click on a definition in the outline, a Definition Editor for that call will be opened in the left-hand pane of the main window.

Notice that in the default Definitions list, most words are in lower case.  As a convention to help read the definitions, I capitalize built-in terms like FRACTION or P_C.  As noted elsewhere, the program is not case sensitive, so you do not have to follow this convention if you prefer to write your definitions differently.  The exact capitalization is only used in displaying the name of a call back to you.
